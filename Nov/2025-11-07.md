### 🚀 Today I Learned

- 네트워크 병목이란?

  - 특정 구간(네트워크/서버/DB)의 처리 한계 때문에 요청이 쌓이고 성능이 저하되는 현상
  - 대규모 트래픽(티켓팅/세일 등)에서 짧은 순간 폭발적인 동시 요청이 원인

- 병목은 없애는 게 아니라 서버까지 도달하기 전에 흩어지고, 줄이고, 늦추는 게 핵심이구나! → 요청 자체를 줄이거나, 서버가 받을 동시 처리량을 조절하거나,
  DB까지 내려가는 깊은 작업을 최소화해야 함

- 간단하게 전략 요약
  | 기술 | 요약 |
  | -------- | ------------------------- |
  | 캐싱 | 반복 요청을 캐시가 대신 응답 → 서버/DB로 트래픽 미도달 |
  | CDN | 전 세계 버전의 캐시 서버 → 원본 서버 보호 |
  | Rate Limiting | 서버가 감당 가능한 속도만 허용하고 429로 차단|
  | Anycast | 같은 IP를 여러 곳에서 운영 → 트래픽이 가까운 POP으로 분산 |

- 티켓팅 할떄 대기열도 병목완화를 위한건데, 대기열조차도 튕기는건 뭘까?
  - 대기열도 하나의 서버 시스템이기 때문에 처리 가능한 연결/세션에 한계가 존재함
  - 즉, 병목은 서버 → 대기열 시스템으로 위치만 이동했을 뿐
  - 근본 해결은 캐싱 + CDN + Rate Limiting + DB 부하 분산의 다층 구조 설계가 필요함
