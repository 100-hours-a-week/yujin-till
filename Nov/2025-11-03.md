### 🚀 Today I Learned

#### 운영체제 & 리눅스

- 운영체제 : 하드웨어와 소프트웨어를 관리하고, 사용자의 요청을 처리하는 시스템 소프트웨어로 UI, System Call, Kernel로 이루어져있다.
- 리눅스 : CLI기반의 OS로 사용자가 자유롭게 수정/배포가 가능하다는 특징이 있다.

#### User Mode & Kernel Mode

- User Mode : 브라우저 등 사용자 응용프로그램이 실행되는 환경으로 해당 모드일때는 UI, System Call에만 접근이 가능하다.
- Kernel Mode : 운영체제가 실행되는 환경으로 UI, System Call, Kernel 등 하드웨어 및 시스템 자원에 대한 접근이 가능하다.

#### 메모리 구조와 관리 기법

- 메모리 구조 : 응용프로그램이 실행되게 위해 필요한 메모리 공간을 운영체제가 할당하는 메모리 공간 체계
- 관리 기법
  - 메모리 오버레이 : 제한된 메모리 부분에 필요한 기능만 로드하여 실행하는 방식으로 초기 컴퓨터 환경에서 등장한것이라 현재는 잘 사용되지 않음
  - 메모리 스왑 : 현재 필요하지 않은 기능을 스왑 공간에 넣어놓고, 현재 필요한 기능을 메모리에 로드하여 관리하는 기법.

#### 프로세스, 스레드

- 프로세스 : 컴퓨터에서 계속 실행되고 있는 프로그램
- 스레드 : 프로세스내에서 병려러리를 위해 생성되는 흐름의 단위

#### 동시성과 병렬성

- 동시성 : 하나의 CPU내에서 물리적 동시 실행이 아닌 동시에 실행되는 것처럼 보이게 하는 개념
- 병렬성 : 여러 CPU 또는 프로세서를 이용하여 물리적으로 동시에 실행하는 개념

#### 스케줄러

- CPU와 같은 시스템 자원을 효율적으로 사용하기 위해 프로세스나 스레드의 실행 순서를 관리하는 것

#### 공유 자원, 임계 영역

- 공유자원 : 여러 사용자, 프로세스, 스레드가 동시에 사용하는 시스템 자원.
- 임계영역 : 공유자원에 접근하는 코드의 일부분

#### 동기화

- 공유자원에 접근하는 여러 작업들때문에 발생할 수 있는 문제를 방지하여 일관성을 보장하기 위한 기술

#### 락

- 하나의 스레드(or 프로세스)가 자원을 사용할때 다른 스레드(or 프로세스)가 접근하지 못하게 잠그는 기술

#### 교착상태

- 스레드(or 프로세스)들이 서로가 락을 풀때까지 무한대기 상태에 빠지는 상태

#### 기아상태

- 특정 스레드(or 프로세스)가 원하는 자원을 얻지 못해 오랫동안 실행되지 못하는 상태

#### 세마포어

- 해당 공유자원에 최대 몇개의 스레드(or 프로세스)를 허가할건지를 관리하는 방식
