### 🚀 Today I Learned

#### [네트워크 용어]

##### Symmetric NAT, Cone NAT의 차이

- Symmetric NAT : 내부 IP + Port 가 같더라도, 목적지가 다르면 계속 NAT 변환을 새롭게함
- Cone NAT: 내부 Ip + Port 가 동일하면, 목적지가 달라도 동일한 NAT 변환 값을 재사용함

##### Port forwarding

- 특정 포트로 들어오는 외부 트래픽을 내부 네트워크의 특정 장치나 포트로 전달하는 기술
- NAT의 일종으로 공인 IP에서 사설 IP간 트래픽 연결
- 사용이유 : 외부 네트워크에서 내부 네트워크 자원에 안전하게 접근할 수 있도록 경로를 제공하기 위함
- NAT vs Port forwarding
  - NAT는 시작점이 내부에서 외부로 통신하고 싶어서 나갈때라 시작점이 내부망(우리서비스)가 먼저임
  - 포트포워딩은 시작점이 외부임. 외부유저가 내부의 어딘가로 요청할때임
  - 라우터의 NAT기능은 자동으로 설정되는데, 포트포워딩은 많은 노드들중에 어디로 가게할 것인지 직접 설정해줘야함
- 장점
  - 트래픽 제어 : 특정 포트에 대해서만 트래픽을 허용 → 불필요한 접근 차단 가능 → 보안강화 가능
  - 세분화된 관리 : 하나의 공인 IP 주소를 사용하면서 여러 내부 장치들을 세분화할 수 있음
  - NAT 규칙의 수동 설정 : 자동 NAT 규칙 대신 사용자가 원하는대로 구성하여 제어 가능

##### VPN

- 사설망 외부에 있는 유저가 사설망 내부에 있는거처럼 연결하는 기술
- 공인 네트워크(인터넷)를 통해 안전한 연결을 제공함
- 외부유저가 VPN과 통신하는 내용을 전체 암호화 = 터널링
- 동작방식 : VPN에 연결하면 외부 유저의 요청을 VPN이 받음 → VPN이 외부유저가 요청한 정보를 얻기 위해 다른 장비에게 요청을 대신 전달해줌 → VPN이 값받음 → 그걸 다시 암호화해서 외부 유저에게 전달
- 우회할때 제일 많이 씀
- 장점 : 프라이버시 보호 강화, 편리한 우회
