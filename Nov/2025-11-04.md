#### [네트워크 장비]

##### NIC (Network Interface Card)

- 컴퓨터, 서버, 또는 기타 네트워크 장치가 **네트워크에 연결**되도록 해주는 하드웨어 구성 요소
- 디바이스 안에 넣는 칩인데, 네트워크 통신을 담당
- **MAC주소**를 고유식별자로 가지고 있어 네트워크 상에서 장치를 구별

##### MAC 주소 (Media Access Control Address)

- 네트워크 장치에 물리적 주소를 나타내는 고유 식별자
- NIC에 하드웨어적으로 내장되어있고, 유일하게 설정된 식별자임
- IP vs MAC 주소
  - MAC주소는 네트워크 카드에 부여되는 영구적인 물리적 주소고, IP는 네트워크상에서의 주소라 바뀔수 있음

##### 리피터

- 네트워크 신호 증폭 장치 → 더 먼거리로 전달하기 위함
- TCP/IP 네트워크 액세스 계층에서 활용되며, 물리 계층에서 작동

##### 허브

- 네트워크에서 데이터를 여러 장치로 전달하는 장치
- 특징
  - 일반적으로 LAN에서 사용
  - LAN에 연결되어있는 노드들(pc,핸드폰)
  - 전달된 모든 트래픽을 연결되어있는 모두에게 전달함 → 트래픽이 많으면 안됨 → 왜? 모든 요청을 브로드 캐스트 하니까
    - 간단한 LAN, 작은회사, 동아리 등에서 사용
    - 구성이 간단해서 저렴함
    - 자신이 받을 패킷이 아니면 버림
- 왜 굳이 브로드 캐스트를 하는 허브를 쓸까?
  - 구현이 간단하고, 저렵하니까 → 정해진 장치한태만 보내는게 무조건적으로 좋은것은아님. 작업하려는 상황에 맞게 어떤 장치를 연결할지 결정하면 됨

##### 스위치

- 네트워크 내에서 데이터 패킷을 장치 간에 전달하는 네트워크 장치
- LAN에 연결되어있는 장치 중 목적지 장치로 정확하게 전달함

#### 네트워크 용어

##### 포트

- 네트워크에서 데이터가 들어오거나 나가는 논리적 또는 물리적 연결 지점

##### BroadCast

- 네트워크 상에서 한 노드가 전송한 데이터가 동일 네트워크에 연결된 모든 노드에 전달되는 통신 방식
- ARP(Address Resolution Protocol) : IP주소를 사용하여 목적지 IP주소에 매핑된 MAC주소를 알아내는 프로토콜
  - 노드가 데이터 전송을 위해서는 대상 장치의 MAC주소 알아야함 → 해당 주소 모를경우 APR Request 메시지를 브로드캐스트로 네트워크에 전송 → 메시지를 받은 IP주소만 자신의 MAC주소를 포함하여 응답 → MAC주소 확인 후에는 유니캐스트로 통신

##### **NAT**

- 주소지를 변환하는 기술
- 사설 IP 주소를 공인 IP 주소로 변환하여 인터넷 통신을 가능하게 하는 기술
- 라우터나 방화벽에서 동작함
- IPv4 주소공간 부족문제를 해결하기 위해 나온 기술임
- ex) 와이파이에 접속하면 와이파이는 공인 ip를 가지고, 해당 와이파이 접속한 기기들은 사설ip를 가짐
- 사용 이유 : **여러 장치가 하나의 공인 IP 주소를 공유하여 인터넷에 접속할 수 있도록 하기 위함**

##### Symmetric NAT

- 동일한 내부 IP와 포트를 사용해도 외부 대상에 따라 서로 다른 매핑된 IP와 포트를 생성하며, 비대칭적 통신을 제한하는 네트워크 주소 변환(NAT) 방식
- 외부에서 연결이 오면 NAT 테이블에 매핑된 정보를 저장하고, 내부 중 어떤 기기인지를 찾을 수 있음 → 목적지 마다 서로 다른 매핑을 생성함
- 외부에서 NAT 장치를 통해 내부로 접근하려면 NAT가 생성한 정확한 매핑 정보(외부 IP와 포트)가 필요하며, 이를 모를 경우 접근이 제한

##### Cone NAT

- 내부 네트워크의 장치와 외부 네트워크 간의 통신에서, 특정 조건에 따라 외부에서 내부로의 접근을 허용하는 네트워크 주소 변환(NAT) 방식
- 허용 범위에 따라
  - Full Cone NAT
    - 내부에서 외부로 나갈때 바뀌는 ip주소를 목적지뿐 아니라 다른 외부 IP들이 알면 같은 ip로 내부한테 데이터 보낼 수 있음
    - NAT 방식 중 가장 개방적인 형태임
    - 실시간 통신에서 연결 효율성이 높지만, 보안성이 상대적으로 낮음
  - Restricted Cone
    - Full Cone NAT 보안문제 개선한 방식
    - 데이터를 보낸 외부 목적지와만 통신이 가능함 + 포트 신경 안씀
  - Port Restricted Cone
    - Restricted Cone의 접근 조건을 더 제한
    - 데이터를 보낸 외부 목적지와만 통신이 가능함 + 포트 매칭 필요

#### [AWS-EC2]

- EC2 : Elastic Compute Cloud의 줄임말로, AWS에서 제공해주는 가상 컴퓨터 본체
- EC2의 구성
  - 인스턴스 : 클라우드상의 가상 서버(가상 컴퓨터 본체)
  - EBS : Elastic Block Storage. 클라우드에서 사용하는 가상 하드디스크를 지칭
    - EC2와 별개로 동작가능함
  - AMI : Amazon Machine Image. EC2 인스턴스를 실행하기 위한 정보를 담고 있는 이미지(템플릿)
- EIP : Elastic IP Address. 고정적인 IP주소를 할당하기위해 인스턴스에 연결하는 서비스
- EC2 접속 방법 비교
  - EC2 Instance Connect 접속 : AWS가 별도의 임시 공개키를 인스턴스에 업로드해서 접속하는 방법, 내 인스턴스가 그런 접속을 허용하도록 설정되어 있어야함
  - 터미널 SSH : .pem은 AWS가 인스턴스 생성할 때 등록한 공개키와 짝을 이루는 개인키,
    SSH가 그걸 풀어서 정상 인증된것
